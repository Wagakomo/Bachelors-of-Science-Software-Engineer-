CS-210 Portfolio Project – Corner Grocer
Project Summary

This project is a grocery item tracking program developed in C++. The program reads a text file containing purchased grocery items and calculates how many times each item appears. It allows the user to search for the frequency of a specific item, display the frequency of all items, and generate a histogram showing each item's purchase count. It also creates a backup file containing the frequency data. This project demonstrates the use of file input/output, maps, functions, and object-oriented programming to solve a practical data analysis problem.

What I Did Well

One of my strengths in this project was organizing the program into separate functions and classes. This made the code easier to understand, maintain, and debug. I also successfully implemented file processing and the use of a map to count item frequencies. The menu-driven interface allows users to easily navigate through the program and access each feature.

Areas for Improvement

If I were to improve this project, I would add stronger input validation to prevent invalid menu selections and incorrect user input. I would also make item searches case-insensitive so users could type item names without worrying about capitalization. Another improvement would be adding the ability to sort grocery items alphabetically or by purchase frequency. These changes would improve the usability, efficiency, and overall user experience.

Challenges and How I Overcame Them

The most challenging part of this project was working with file input/output and using a map to store and count grocery items. Creating the histogram and ensuring the output matched the project requirements also required careful testing. I overcame these challenges by breaking the project into smaller tasks, testing each function individually, and using Visual Studio's debugger to locate and fix errors. The course materials, zyBooks, instructor resources, and ChatGPT also helped me better understand concepts that I found difficult.

Maintainability, Readability, and Adaptability

I made this program maintainable by separating tasks into individual functions and keeping each function focused on a single responsibility. I used descriptive variable names, consistent indentation, and comments to improve readability. This modular design allows new features, such as additional reports or different sorting options, to be added with minimal changes to the existing code, making the program adaptable for future enhancements.
